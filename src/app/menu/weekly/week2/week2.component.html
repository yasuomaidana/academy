The scientific method

How to go through the process?

When you are in charge, it is relevant to know the entire process of whatever you are doing. Because that way, you could participate in any stage and understand the value that others provide.

Sometimes, it is better to do something partially right than not do anything at all.

Journaling

It helps you to keep track of your process and to consult later.

Important journal questions

  • What are you
    • Thinking?
    • Doing?
  • What do you want to
    • Do?
    • Know?
  • What did you discover?

Problem discovery

This process is about discovering the components of a problem, their manifestations, and understanding their causes and effects.

Observation

You must avoid taking what you know for granted. First, observe the system, and try to understand it.

Data gathering

Try to identify cause-effect relationships and then gather information about them.

Data analysis

Try to use the data to prove your ideas wrong or right. You must analyze data, even if you think you understand the problem.

Hypothesis generation

Your ideas should be written down in a sentence that explains what is going on.

You could use the Taguchi diagram to externalize and organize ideas.

Experimental design

Experimental design is a methodology that helps you be efficient and allows you to have a degree of reliability on the results you get.

Tests

These are a set of protocols that anyone can use to replicate your observations. They are the proof of your conclusion.

Problem definition

You should define your problem in a single simple sentence. However, you could have several. Sometimes, the nature of the problem guides you to multiples phenomena.

Definition

The definition should be a single sentence that explains the problem. It must have coherence with your data and work.

Constraints

Make sure to list all the conditions and their importance. This will allows others to replicate your work.

Search for solution

Research

Start by looking at what others have done. Then, if nobody has solved your hypothesis, you could start yours. The advantage of doing this way is that you get the wise of others.

Scientific papers

The standard to share discoveries is the scientific paper. They show the methodology followed, the results, and conclusions.

Brainstorming

Think about possibilities and write them. It does not matter if those do not make sense at all.

Modeling

Discard everything that does not work with the problem definition

Prototyping

It is a simplified system that proves if the component can be built.

Specification

It is the guide with which you build the solution.

The missing semester: Course overview + the shell

Commands allow you to do several kinds of stuff some of them are not possible through graphical interfaces.

  • "echo" with $PATH shows you the path variables, echo is a kind of print.
  • Absolute paths are paths that fully determine the location of a file.
  • "PWD" prints the current path you are in
  • cd changes directory
  • cd ~ returns you to the home directory
  • cd - returns you to the previous directory you were
  • ls show you the files and directories
  • Command + —help usually gives you help or small documentation about the command.
  • mv used to move/change name arguments original path, a new path
  • cp copy command
  • rm removes files, if you want to remove a directory with files is rm -r, to delete a directory is rmdir. To create a directory use mkdir
  • man gives the manual page of any program
  • ctr l allows you to clear the console
  • command < pathfileInput > pathFileOutput
  • command > pathFileOutput
  • cat print content of a file
  • >> used for append
  • | (pipe) takes the output of a program and use it as an argument of the next
  • tail -nX prints the nX last inputs
  • Example
  • ls -l | tail -n2
  • curl --head --silent google.com
  • grep -i content-length
  • sudo su convert the shell into superuser
  • tee writes into a file
  • xdg-open fileNamePath opens a file (linux)
