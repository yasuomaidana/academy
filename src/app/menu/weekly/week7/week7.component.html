Agile in Practice

Pair programming means work together at the same time, even on the same computer. It is done to improve the quality of the code and to expand its knowledge of it.

Agile involves continuous integration. This minimizes error probability. In every iteration, the team looks to give value while interacts with others teams. These interactions ease feedback.

The done definition is relevant to agile management because it tells you how to measure your progress.

To say something is done, it should:

  • Be signed by the customer

  • Have enough acceptance criteria and assumptions

  • Be reviewed by another developer

  • Be successfully implemented

  • Have user experience test

  • Have been released in a way that won’t need further modifications

Test-driven development advantages

  • Ensures quality
  • Keeps code clear, simpler and testeable
  • Provides documentation for different team members
  • Repeatable tests
  • Enables rapid changes
